Confluence Protractor Base

The Confluence® Protractor Base is a simple solution to implement und run UI tests for Atlassian® Confluence®. It provides basic functionality like creating pages, spaces, users and groups as well as login, authenticate as admin and open pages.

Local setup

To start up local UI tests, you need the following setup

  • protractor
  • a running Confluence® instance

Steps to run UI tests

There has to be UI tests in your project

  1. Start a clean Confluence® docker instance
    • The instance should be locally available under http://confluence:8090\ (!) Note: There may be changes needed in the /etc/hosts file 127.0.0.1 confluence
  2. Setup required resources
    1. Run npm install in your plugin project, after that, there should be runnable programs inside node_modules/.bin
    2. Update webdriver-manager once with node_modules/.bin/webdriver-manager update
    3. Start webdriver-manager in another console using node_modules/.bin/webdriver-manager start
  3. Run UI tests with an npm script using npm run test-e2e (e2e means End-to-End)
